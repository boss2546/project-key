

# MVP v0.1 — Locked Spec

## 0) Product Definition

> **Private knowledge space ที่รับไฟล์สำคัญเข้ามา จัดให้เป็นระบบ สร้างไฟล์สรุป `.md` และเปิดให้ผู้ใช้นำข้อมูลนั้นไปใช้กับ AI ได้อย่างควบคุมได้**

นี่คือเวอร์ชันที่เล็ก แต่ยังตรง Vision เพราะไม่ใช่แค่ chat with files แต่เป็น **พื้นที่ข้อมูลส่วนตัวที่จัดระบบแล้วพร้อมใช้กับ AI** 

---

# 1) Scope ที่ล็อกแล้ว

## A. Ingestion

ระบบต้องรับไฟล์ที่มั่วและไม่มีโครงสร้างเข้ามาได้

### รองรับไฟล์รอบแรก

* PDF
* TXT
* MD
* DOCX

### สิ่งที่ระบบต้องทำ

* อัปโหลดหลายไฟล์
* เก็บไฟล์ต้นฉบับ
* extract text
* เก็บ metadata พื้นฐาน
* ผูกไฟล์กับ user owner

---

## B. Organization

ระบบต้องจัดข้อมูล 3 อย่าง

### 1. Grouping

ตอบให้ได้ว่าไฟล์ไหนอยู่เรื่องเดียวกัน

### 2. Importance Scoring

ตอบให้ได้ว่าไฟล์ไหนสำคัญกว่า

### 3. Markdown Summary Generation

สร้างไฟล์ `.md` ของแต่ละไฟล์เพื่อเป็น AI-ready layer

---

## C. AI Use

ผู้ใช้ถามในแชทได้ และระบบต้องเลือกข้อมูลที่เหมาะสมไปใช้

### Representation ที่ระบบเลือกได้

* summary `.md`
* excerpt
* raw extracted text

รอบแรกยัง **ไม่ต้องส่ง binary raw file ตรงเข้า model** เป็น feature หลัก
ให้เริ่มจาก text path ก่อน เพื่อให้ระบบนิ่งและเร็ว

---

# 2) Product Principles ที่ล็อกแล้ว

เพื่อให้ตรง Vision เราจะยึด 4 หลักนี้:

### 1. Private by default

ทุกข้อมูลเป็นของ user คนนั้น และมองเห็นเฉพาะของตัวเอง

### 2. Original file is preserved

เก็บไฟล์ต้นฉบับไว้ ไม่ทำลาย

### 3. Organized by system

ระบบต้องช่วยจัด ไม่ใช่ให้ user จัดเองหมด

### 4. AI use must be explainable

ต้องบอกได้ว่า AI ใช้ไฟล์/สรุปอะไรตอบ

ทั้งหมดนี้สอดคล้องกับแกน Vision เรื่อง preservation, privacy, structure และ seamless use

# 4) User Flow ที่ล็อกแล้ว

## Flow 1: Upload

ผู้ใช้อัปโหลดไฟล์หลายไฟล์เข้ามาในพื้นที่ส่วนตัว

## Flow 2: Process

ระบบ:

* เก็บ raw file
* extract text
* สร้าง metadata

## Flow 3: Organize

ระบบ:

* จัดกลุ่มไฟล์
* ให้คะแนนความสำคัญ
* สร้าง `.md summary`

## Flow 4: Browse

ผู้ใช้เปิดดูข้อมูลที่จัดแล้วได้

## Flow 5: Ask AI

ผู้ใช้ถามคำถาม

## Flow 6: Retrieve

ระบบเลือก cluster/file ที่เกี่ยว

## Flow 7: Answer

ระบบส่ง summary / excerpt / raw text ให้ AI และแสดง source ที่ใช้

# 7) Spec ของไฟล์ `.md` ที่ล็อกแล้ว

ทุกไฟล์ที่ประมวลผลเสร็จต้องมี summary markdown 1 ไฟล์

ตัวอย่าง schema:

```md
---
file_id: <id>
original_filename: <filename>
filetype: <pdf/docx/txt/md>
cluster: <cluster title>
importance_score: <0-100>
importance_label: <high/medium/low>
is_primary_candidate: <true/false>
uploaded_at: <timestamp>
---

# Summary
สรุปเนื้อหาหลักของเอกสารนี้

# Key Topics
- ...
- ...
- ...

# Key Facts
- ...
- ...
- ...

# Why This File Matters
- ...
- ...

# Suggested Usage
- ใช้ตอน...
- ควรใช้เป็น summary / excerpt / raw
```

# 8) Logic ที่ล็อกแล้ว

## A. Grouping Logic

รอบแรกใช้หลัก:

* semantic similarity จาก extracted text
* filename hints
* shared keywords/topics

ผลลัพธ์:

* ไฟล์อยู่ cluster เดียวกัน
* ตั้งชื่อ cluster แบบง่าย

---

## B. Importance Scoring Logic

รอบแรกคิดคะแนนจาก:

* ความครบของเนื้อหา
* ความยาวที่มีสาระ
* ความชัดของชื่อไฟล์
* ความใหม่
* ความเป็นไฟล์หลัก/สรุป/เวอร์ชันสุดท้าย

ผลลัพธ์:

* score 0–100
* label high / medium / low
* primary candidate 1 ตัวต่อ cluster ถ้าเป็นไปได้

---

## C. Retrieval Logic

เวลาผู้ใช้ถาม:

1. หา cluster ที่เกี่ยว
2. เลือกไฟล์เด่นใน cluster
3. ตัดสินใจ mode

### Mode rules รอบแรก

* ใช้ `.md summary` เป็น default
* ใช้ excerpt เมื่อคำถามเจาะเฉพาะส่วน
* ใช้ raw extracted text เมื่อจำเป็นต้องละเอียด

# 9) Open-source Stack ที่ล็อกแล้ว

## Core Direction

เรา **ไม่ใช้ Logseq เป็นฐานหลักแล้ว**
แต่จะ **สร้าง web app ของเราเอง** และใช้โอเพ่นซอร์สเป็นส่วนประกอบ

## Recommended Stack

### Frontend

* Next.js

### Backend

* FastAPI

### Document Parsing

* Docling เป็นตัวหลัก

### Chat / Retrieval Reference

* ใช้แนวคิดจาก Document Chat System
* retrieval architecture อ้างอิงแนว RAGFlow

### Storage

* object storage สำหรับ raw files
* Postgres สำหรับ metadata
* vector store แบบเบาสำหรับ similarity search

### LLM Use

* ใช้ model สำหรับ

  * summarize
  * cluster naming
  * answer generation

# 11) Success Criteria ที่ล็อกแล้ว

MVP นี้ถือว่าใช้ได้ ถ้า user ทำ 4 อย่างนี้ได้จริง:

* อัปโหลดไฟล์มั่วๆ ได้
* เห็นว่าไฟล์ไหนอยู่เรื่องเดียวกัน
* เห็นว่าไฟล์ไหนสำคัญกว่า
* ถาม AI แล้วระบบเลือกข้อมูลที่เกี่ยวให้ได้โดยไม่ต้องหาไฟล์เอง

# 12) One-line MVP Statement

> **Store → Organize → Reuse with AI**

อันนี้คือหัวใจของ MVP ใหม่ และตรงกับ Vision มากกว่าแบบเดิมที่เป็นแค่ upload แล้วถาม AI

# 13) ฟันธงสุดท้าย

## เราล็อกแล้วว่า:

* เริ่มจาก **private personal data space**
* ไม่เริ่มจาก note app
* ไม่เริ่มจาก graph
* ไม่เริ่มจาก plugin ecosystem
* differentiation ของเราคือ
  **grouping + importance + `.md` summary + controlled AI use**

นี่คือแผนที่ **เร็วพอจะเริ่มทำ** และ **ไม่หลุดจาก Vision**
