### 1.1 สิ่งที่ต้องมี

| รายการ | รายละเอียด |


## 2. หน้า My Data — จัดการไฟล์

> เป็นหน้าแรกที่เปิดขึ้นมา ใช้สำหรับ **อัปโหลดไฟล์** และ **จัดระเบียบข้อมูล**

![หน้า My Data](docs/img_mydata.png)

### 2.1 อัปโหลดไฟล์

**วิธีที่ 1: ลากวาง (Drag & Drop)**
1. ลากไฟล์จาก File Explorer
2. วางลงในพื้นที่อัปโหลด (กรอบเส้นประ)

**วิธีที่ 2: เลือกไฟล์**
1. คลิกที่พื้นที่อัปโหลด
2. เลือกไฟล์จาก dialog

**ไฟล์ที่รองรับ:**

| ประเภท | นามสกุล | หมายเหตุ |
|--------|---------|---------|
| PDF | `.pdf` | ดึงข้อความอัตโนมัติ |
| Text | `.txt` | ข้อความล้วน |
| Markdown | `.md` | รองรับ formatting |
| Word | `.docx` | ดึงข้อความจากเอกสาร |

> ⚠️ **ขนาดไฟล์สูงสุด:** 20 MB ต่อไฟล์

### 2.2 จัดระเบียบด้วย AI

คลิกปุ่ม **"+ จัดระเบียบด้วย AI"** (มุมขวาบน, สีม่วง) เพื่อเริ่ม AI Pipeline:

```
อัปโหลดไฟล์ → 「จัดระเบียบด้วย AI」→ ผลลัพธ์:

  ✅ จัดกลุ่มไฟล์เป็น Collections
  ✅ ให้คะแนนความสำคัญ (สูง / กลาง / ต่ำ)
  ✅ สร้างสรุป (Summary) ทุกไฟล์
  ✅ เพิ่ม Metadata (tags, sensitivity, freshness)
  ✅ สร้าง Knowledge Graph อัตโนมัติ!
```

> ⏱️ **ใช้เวลา:** ประมาณ 30-120 วินาที ขึ้นอยู่กับจำนวนไฟล์

### 2.3 Enrich Metadata

คลิกปุ่ม **"✦ Enrich Metadata"** (มุมขวาบน) เพื่อให้ AI วิเคราะห์ metadata เพิ่มเติม:

- **Tags** — หัวข้อสำคัญของไฟล์
- **Sensitivity** — ระดับความลับ (Normal / Sensitive / Confidential)
- **Freshness** — ความทันสมัย (Current / Aging / Stale)
- **Source of Truth** — เป็นแหล่งข้อมูลหลักหรือไม่

### 2.4 รายการไฟล์

เมื่ออัปโหลดและจัดระเบียบแล้ว จะเห็น:

| ข้อมูลที่แสดง | ตัวอย่าง |
|--------------|---------|
| ชื่อไฟล์ | research_knowledge_graph.md |
| ประเภท | MD |
| สถานะ | ✅ ready / ⏳ processing |
| Importance | 🔴 สูง / 🟡 กลาง / 🟢 ต่ำ |
| Tags | `knowledge graph`, `AI`, `วิจัย` |
| ลบ | ปุ่ม 🗑️ ด้านขวา |

### 2.5 ลบไฟล์

1. คลิก 🗑️ ด้านขวาของไฟล์
2. ยืนยันการลบ
3. ไฟล์จะถูกลบออกจากระบบ (ไม่กู้คืนได้)

## 3. หน้า Knowledge View — สำรวจความรู้

> ดูข้อมูลที่ถูกจัดระเบียบแล้ว แบ่งเป็น 3 แท็บ

![หน้า Knowledge View](docs/img_knowledge.png)

### 3.1 Tab: Collections

แสดง **กลุ่มไฟล์** ที่ AI จัดให้อัตโนมัติ

- แต่ละ Collection มี: **ชื่อกลุ่ม**, **สรุปเนื้อหา**, **จำนวนไฟล์**
- คลิกที่ Collection เพื่อขยายดูรายการไฟล์ภายใน
- แต่ละไฟล์แสดง importance badge (high/medium/low)

**ตัวอย่าง:**
```
📁 ภาพรวมและแผนงานโปรเจกต์ NOVA (3 ไฟล์)
   ├── project_nova_plan.md          🔴 สูง
   ├── meeting_notes_sprint5.md      🟡 กลาง
   └── tech_spec_knowledge_graph.md  🔴 สูง
```

### 3.2 Tab: Notes & Summaries

แสดง **entities ที่ AI ดึงออกมา** จากเอกสาร — เช่น ชื่อคน, โปรเจกต์, องค์กร, concepts

> 💡 Entities จะปรากฏหลังจากสร้าง Knowledge Graph แล้วเท่านั้น

### 3.3 Tab: Context Packs

แสดง **Context Packs** — บริบทระดับสูงที่กลั่นจากหลายไฟล์

#### สร้าง Context Pack ใหม่:
1. คลิก **"+ สร้าง Context Pack"**
2. เลือกประเภท: การเรียน / ทำงาน / โปรเจกต์ / โปรไฟล์
3. ตั้งชื่อ (เช่น "NOVA Research Context")
4. เลือก source files / clusters
5. คลิก **Generate** → AI จะกลั่นเนื้อหาให้

#### ประโยชน์ของ Context Pack:
- ใช้เป็น "บริบทสำเร็จรูป" สำหรับ AI Chat
- ไม่ต้องส่งไฟล์ทั้งหมดให้ AI ทุกครั้ง
- สรุปความรู้ข้ามไฟล์ไว้ในที่เดียว

### 3.4 มุมมอง Cards vs Table

คลิกปุ่มด้านขวาบนเพื่อสลับ:
- **Cards** — แสดงเป็นการ์ดสวย (default)
- **Table** — แสดงเป็นตาราง (กะทัดรัด)

## 4. หน้า Graph — Knowledge Graph

> 🌐 **หัวใจสำคัญของ v3** — สำรวจความเชื่อมโยงระหว่างข้อมูลทั้งหมดของคุณ

![Knowledge Graph แบบ Global](docs/img_graph_global.png)

### 4.1 Global Graph (มุมมองภาพรวม)

แสดง **nodes ทั้งหมด** ในระบบพร้อม edges ที่เชื่อมต่อ

**ความหมายของสี:**

| สี | ประเภท | ตัวอย่าง |
|----|--------|---------|
| 🟡 เหลือง | ไฟล์ (source_file) | research_knowledge_graph.md |
| 🟠 ส้ม | Entity (คน/องค์กร) | ดร.สมชาย วิจัยดี |
| 🔵 ฟ้า | Tag (หัวข้อ) | knowledge graph, AI |
| 🟢 เขียว | Collection (กลุ่ม) | ภาพรวมและแผนงานฯ |
| 🟦 ฟ้าเข้ม | Context Pack | NOVA Research Context |
| 🟣 ม่วง | Person (บุคคล) | วิภา ข้อมูลศาสตร์ |

**ขนาด Node:** ยิ่งใหญ่ = ยิ่งสำคัญ (importance score สูง)

### 4.2 การใช้งาน Graph

#### ซูม & แพน
- **เลื่อน scroll** — ซูมเข้า/ออก
- **ลากพื้นที่ว่าง** — เลื่อนกราฟ
- **ลาก node** — ย้ายตำแหน่ง node

#### คลิก Node → Detail Panel

![Detail Panel](docs/img_graph_detail.png)

คลิกที่ node ใดก็ได้ → **Detail Panel** จะเลื่อนเข้ามาจากด้านขวา แสดง:

| ส่วน | รายละเอียด |
|------|-----------|
| **ประเภท** | badge สี เช่น `SOURCE_FILE`, `TAG`, `ENTITY` |
| **ชื่อ** | ชื่อ node |
| **สรุป** | คำอธิบายสั้น |
| **Metadata** | importance, freshness, node_family |
| **Relations** | รายการ edges ที่เชื่อมต่อ (ทั้ง incoming + outgoing) |

คลิก **✕** หรือคลิกที่อื่นเพื่อปิด panel

### 4.3 ค้นหา & กรอง

#### ช่องค้นหา (Search)
1. พิมพ์คำค้นใน input ด้านซ้ายบน (เช่น "NOVA")
2. Nodes ที่ตรง → **highlight สว่าง**
3. Nodes ที่ไม่ตรง → **จางลง** (opacity ลด)
4. ลบคำค้นเพื่อแสดงทั้งหมดอีกครั้ง

#### Filter Chips (ปุ่มกรองประเภท)
แถบปุ่ม 6 ปุ่มด้านบน:

```
[ ● ไฟล์ ] [ ● Entity ] [ ● Tag ] [ ● Collection ] [ ● Pack ] [ ● Person ]
```

- **คลิกเพื่อ toggle** — ซ่อน/แสดง nodes ประเภทนั้น
- ปุ่ม active = สีเข้ม, ปุ่ม inactive = จาง
- ใช้เพื่อโฟกัสดูเฉพาะ node type ที่สนใจ

**ตัวอย่าง:**
- ต้องการดูเฉพาะ ไฟล์ + Collection → ปิด Entity, Tag, Pack, Person
- ต้องการดูว่ามี Entity อะไรบ้าง → ปิดทุกอย่างยกเว้น Entity

### 4.4 Local Graph (มุมมองรอบ Node)

1. คลิกปุ่ม **"Local"** (มุมขวาบน) เพื่อสลับจาก Global
2. จะแสดงเฉพาะ **node ที่เลือก + เพื่อนบ้าน** ในระยะที่กำหนด
3. ปรับ **Depth** (ระยะ hop) ได้ 1-3:
   - **Depth 1** — เฉพาะ node ที่เชื่อมต่อโดยตรง
   - **Depth 2** — รวมเพื่อนของเพื่อน
   - **Depth 3** — ขยายออกไปอีก 1 ชั้น

> 💡 **เมื่อไรใช้ Local Graph?** เมื่อต้องการโฟกัสดูว่า node หนึ่งเชื่อมโยงกับอะไรบ้าง โดยไม่ถูก nodes อื่นรบกวน

### 4.5 Rebuild Graph

คลิก **"↻ Rebuild Graph"** (มุมขวาบน) เพื่อสร้างกราฟใหม่ทั้งหมด:
- ใช้เมื่อเพิ่มไฟล์ใหม่แล้วอยากอัปเดตกราฟ
- ใช้เมื่อรู้สึกว่ากราฟไม่แม่นยำ
- ใช้เวลาประมาณ 30-60 วินาที

## 5. หน้า AI Chat — ถาม AI พร้อมหลักฐาน

> AI ที่ตอบโดยอ้างอิงจากข้อมูลจริงของคุณ — ไฟล์, profile, context packs, **และ Knowledge Graph**

![AI Chat พร้อม Sources Panel](docs/img_chat.png)

### 5.1 วิธีถาม

1. พิมพ์คำถามในช่องด้านล่าง
2. คลิก ➤ หรือกด Enter
3. รอ AI ตอบ (5-20 วินาที)

**ตัวอย่างคำถามดีๆ:**

| ประเภท | ตัวอย่าง |
|--------|---------|
| 🔍 สรุปโภาพรวม | "สรุปโปรเจกต์ NOVA ให้หน่อย" |
| 👥 เกี่ยวกับบุคคล | "ดร.สมชาย ทำอะไรบ้างในโปรเจกต์?" |
| 🔗 ความเชื่อมโยง | "Knowledge Graph เกี่ยวข้องกับ NOVA ยังไง?" |
| 📋 Action Items | "มีงานอะไรที่ยังไม่เสร็จบ้าง?" |
| 💡 วิเคราะห์ | "เทคโนโลยีหลักที่ใช้ในโปรเจกต์คืออะไร?" |

### 5.2 Injection Badge

ใต้คำตอบ AI จะมี badge แสดงว่า AI ใช้ข้อมูลอะไรบ้าง:

```
🧠 โปรไฟล์ผู้ใช้ + 3 ไฟล์ + 14 graph nodes + 15 relations
```

แสดงให้เห็นว่า AI ใช้ **7 ชั้นข้อมูล** ในการตอบ

### 5.3 Sources Panel (แถบด้านขวา)

![Sources Panel](docs/img_sources.png)

แสดงหลักฐานทั้งหมดที่ AI ใช้ แบ่งเป็น 6 ส่วน:

| ส่วน | แสดงอะไร |
|------|---------|
| 👤 **โปรไฟล์** | ว่าใช้ profile ของคุณหรือไม่ (✅ badge ถ้าใช้) |
| 📦 **Context Packs** | Context Packs ที่ AI ใช้ (ชื่อ + ประเภท) |
| 📄 **ไฟล์ที่ใช้** | รายชื่อไฟล์ที่เป็น source (เช่น project_nova_plan.md) |
| 🔗 **Nodes & Edges** | Graph nodes ที่เกี่ยวข้อง (แสดงเป็น chip สี) |
| 💡 **เหตุผลในการเลือก** | AI อธิบายว่าทำไมถึงเลือก sources เหล่านี้ |
| 📊 **Evidence Graph** | Mini graph แสดงความเชื่อมโยงของ sources |

### 5.4 Profile Indicator

ด้านขวาบนของ header แสดง:
- **● Profile: Active** (สีเขียว) — AI มี profile ของคุณ ตอบได้ตรงสไตล์
- **● Profile: Not set** (สีเทา) — ยังไม่ได้ตั้ง profile

### 5.5 Source Chips

ตรง welcome screen แสดง chips 5 อัน:
```
[ 👤 Profile ] [ 📦 Packs ] [ 📁 Collections ] [ 📄 Files ] [ 🔗 Graph ]
```
แสดงว่าระบบมี data layer ไหนบ้างที่พร้อมใช้

## 6. My Profile — ให้ AI รู้จักคุณ

> ตั้งค่าโปรไฟล์เพื่อให้ AI ตอบได้ตรงกับตัวคุณมากขึ้น

![My Profile Modal](docs/img_profile.png)

### 6.1 เปิด Profile

คลิก **"My Profile"** ที่ด้านล่างของ Sidebar

### 6.2 กรอก 5 ฟิลด์

| ฟิลด์ | คำอธิบาย | ตัวอย่าง |
|-------|---------|---------|
| **ตัวตน** | คุณเป็นใคร ทำอะไร | "นักศึกษาปริญญาโท สาขา AI" |
| **เป้าหมาย** | กำลังทำอะไร ต้องการอะไร | "วิจัยเรื่อง Knowledge Graph สำหรับ PKM" |
| **สไตล์การทำงาน** | ทำงานแบบไหน | "ชอบเห็นภาพรวมก่อน ค่อยลงรายละเอียด" |
| **รูปแบบ Output** | ต้องการคำตอบแบบไหน | "ตอบเป็นข้อๆ กระชับ มี action items" |
| **บริบทเพิ่มเติม** | ข้อมูลอื่นที่เกี่ยวข้อง | "ทำงานที่ lab AI ของมหาวิทยาลัย" |

### 6.3 บันทึก

คลิก **"💾 บันทึกโปรไฟล์"** → จุดสถานะที่ sidebar จะเปลี่ยนเป็น 🟢

### 6.4 ผลของ Profile ต่อ AI

เมื่อตั้ง Profile แล้ว:
- AI จะ **เข้าใจบริบท** ของคุณตั้งแต่แรก
- AI จะ **ตอบในสไตล์** ที่คุณกำหนด
- ไม่ต้อง **อธิบายตัวเอง** ทุกครั้งที่ถาม
- `injection_summary` จะแสดง "โปรไฟล์ผู้ใช้" เป็น Layer แรก

## 7. Flow การใช้งานแนะนำ

### 📋 First-time Setup (ครั้งแรก)

```
1️⃣  ตั้งค่า Profile
     └→ คลิก My Profile → กรอก 5 ฟิลด์ → Save

2️⃣  อัปโหลดไฟล์
     └→ หน้า My Data → ลากวางไฟล์ (PDF, TXT, MD, DOCX)

3️⃣  จัดระเบียบ + สร้างกราฟ
     └→ คลิก "จัดระเบียบด้วย AI" → รอ 30-120 วินาที
     └→ ระบบจะ: cluster → score → summarize → enrich → graph build

4️⃣  สำรวจ Knowledge Graph
     └→ หน้า Graph → ดูความเชื่อมโยง → คลิก node ดู detail

5️⃣  ถาม AI
     └→ หน้า AI Chat → ถามคำถาม → ดู Sources Panel ด้านขวา
```

### 🔄 Daily Use (ใช้งานประจำ)

```
เพิ่มไฟล์ใหม่     → My Data → อัปโหลด → จัดระเบียบ (auto rebuild graph)
สำรวจข้อมูล        → Knowledge View → เลือก tab ที่สนใจ
ดูความเชื่อมโยง    → Graph → ค้นหา/กรอง → คลิก node ดู detail
ถาม AI            → AI Chat → พิมพ์คำถาม → ดูคำตอบ + sources
อัปเดต profile    → My Profile → แก้ไข → Save
```

### 🧠 Advanced Use (ใช้งานขั้นสูง)

```
สร้าง Context Pack → Knowledge View → Packs tab → + สร้าง Pack
Rebuild Graph      → Graph → Rebuild Graph (เมื่อเพิ่มไฟล์ใหม่)
Enrich Metadata    → My Data → Enrich Metadata (เมื่อต้องการ tags เพิ่ม)
ดู Local Graph     → Graph → คลิก node → สลับ Local → ปรับ depth
```

## 8. คำถามที่พบบ่อย (FAQ)

### ❓ ระบบรองรับไฟล์ภาษาไทยไหม?

✅ **รองรับเต็มที่** — ทั้ง ไฟล์ภาษาไทย, AI ตอบเป็นภาษาไทย, tags ภาษาไทย, entities ภาษาไทย

### ❓ อัปโหลดได้กี่ไฟล์?

ไม่จำกัดจำนวน แต่แนะนำ **ไม่เกิน 50 ไฟล์** ในครั้งเดียว เพื่อให้ AI pipeline ทำงานได้ราบรื่น

### ❓ ต้อง Rebuild Graph ทุกครั้งที่เพิ่มไฟล์ไหม?

**ไม่จำเป็น** — เมื่อคุณคลิก "จัดระเบียบด้วย AI" ระบบจะ rebuild graph อัตโนมัติ
จะ manual rebuild เฉพาะเมื่อต้องการ refresh graph ที่ไม่ได้ผ่านการ organize

### ❓ Knowledge Graph สร้างจากอะไร?

AI วิเคราะห์ **summaries ของทุกไฟล์** แล้วดึง:
- **Entities** — ชื่อคน, โปรเจกต์, องค์กร, เทคโนโลยี
- **Tags** — หัวข้อ/คีย์เวิร์ดสำคัญ
- **Relations** — ความเชื่อมโยงระหว่าง entities (เช่น file mentions entity)

### ❓ AI ตอบช้ามาก ทำยังไง?

- ตรวจสอบ **API Key** ว่ายังใช้ได้อยู่
- ตรวจสอบ **inet** — ระบบต้องเชื่อมต่อ OpenRouter API
- ลองถามคำถามสั้นๆ ก่อน

### ❓ ลบไฟล์แล้ว Graph จะอัปเดตเองไหม?

ยังไม่อัปเดตอัตโนมัติ — ต้อง **Rebuild Graph** หลังจากลบไฟล์

### ❓ ข้อมูลเก็บไว้ที่ไหน?

| ข้อมูล | ตำแหน่ง |
|--------|---------|
| ไฟล์ดิบ | `uploads/` |
| Summaries | `summaries/` |
| Context Packs | `context_packs/` |
| ฐานข้อมูล | `pdb.db` (SQLite) |
| API Key | `.env` |

ทุกอย่างอยู่ในเครื่องของคุณ — **ไม่มีข้อมูลถูกส่งไปเก็บที่อื่น** (ยกเว้นเรียก AI ผ่าน API)

### ❓ รีเซ็ตข้อมูลทั้งหมดได้ไหม?

ได้ — ใช้คำสั่ง API:
```bash
curl -X DELETE http://localhost:8000/api/reset
```
⚠️ **คำเตือน:** จะลบข้อมูลทั้งหมดรวมถึง graph, files, profile, packs

## 9. เทคนิคการใช้งานขั้นสูง

### 9.1 ใช้ Graph ค้นหาความเชื่อมโยงที่ซ่อนอยู่

1. เปิด **Global Graph**
2. มองหา **nodes ที่อยู่ตรงกลาง** → นั่นคือ nodes ที่เชื่อมโยงมากที่สุด
3. คลิก node นั้น → ดู Detail Panel → อ่าน relations
4. สลับเป็น **Local Graph** → ดูเฉพาะ neighborhood ของ node นั้น

### 9.2 ใช้ Filter เพื่อวิเคราะห์

| สิ่งที่อยากดู | Filter ที่ควรเปิด |
|-------------|-----------------|
| ภาพรวมไฟล์ | ✅ ไฟล์ + ✅ Collection |
| หัวข้อทั้งหมด | ✅ Tag เท่านั้น |
| บุคคลในข้อมูล | ✅ Person + ✅ Entity |
| ทุกอย่าง | เปิดหมด |

### 9.3 ถาม AI อย่างมีประสิทธิภาพ

**❌ คำถามไม่ดี:**
- "บอกข้อมูลมาหน่อย" (กว้างเกินไป)
- "อะไรก็ได้" (ไม่มี focus)

**✅ คำถามดี:**
- "สรุปโปรเจกต์ NOVA แบบ bullet points"
- "ดร.สมชาย มีบทบาทอะไรบ้าง ในแต่ละไฟล์?"
- "เปรียบเทียบ tech spec กับ meeting notes — มีอะไรที่ไม่ตรงกัน?"
- "ใน meeting notes มี action items อะไรที่ยังไม่เสร็จ?"

### 9.4 Context Pack Strategy

| เมื่อไรควรสร้าง | ตัวอย่าง |
|---------------|---------|
| มีไฟล์เยอะในหัวข้อเดียว | Pack "สรุปวิจัย Knowledge Graph" จาก 5 ไฟล์ |
| เตรียมตอบคำถามเฉพาะทาง | Pack "NOVA Project Overview" สำหรับ demo |
| ต้องการให้ AI เข้าใจบริบทลึก | Pack "My Research Background" จากไฟล์วิจัยทั้งหมด |

### 9.5 ดู Sources Panel อย่างละเอียด

เมื่อ AI ตอบ ให้ดู Sources Panel ทางขวาเสมอ:

1. **ตรวจ Profile** — ถ้าไม่ active ให้ไปตั้งค่า
2. **ตรวจ Files** — AI ใช้ไฟล์ที่เกี่ยวข้องจริงไหม?
3. **ตรวจ Nodes & Edges** — AI เห็นความเชื่อมโยงไหม?
4. **อ่าน Reasoning** — AI อธิบายเหตุผลการเลือกข้อมูลชัดเจนไหม?

ถ้าพบว่า AI ตอบไม่ตรง → ลอง:
- ถามใหม่โดยระบุชื่อไฟล์หรือหัวข้อให้ชัดเจนขึ้น
- สร้าง Context Pack ที่เกี่ยวข้องเพิ่ม
- Rebuild Graph เพื่ออัปเดตความเชื่อมโยง
