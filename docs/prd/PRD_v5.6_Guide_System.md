## 2. User Journey Maps — 3 เส้นทาง

### Journey A: ผู้ใช้ใหม่ (Onboarding)
```
สมัครสมาชิก
  ↓
เข้าเว็บครั้งแรก → [คู่มือเปิดอัตโนมัติ + แสดง Checklist]
  ↓                    ☐ อัปโหลดไฟล์แรก
  ↓                    ☐ จัดระเบียบด้วย AI
  ↓                    ☐ ลองถาม AI แชท
  ↓
กดปิดคู่มือ → ลองทำตาม Checklist
  ↓
✅ "อ๋อ ระบบทำแบบนี้เอง!"
```

### Journey B: ผู้ใช้อยากเชื่อมต่อ MCP
```
กดปุ่ม 📖 คู่มือ → แท็บ "เชื่อมต่อ"
  ↓
เลือก Claude Desktop → เห็น 4 ขั้นตอน
  ↓
กด "คัดลอก Config" → วางใน Claude → รีสตาร์ท
  ↓
กด "ทดสอบ" → ✅ เชื่อมต่อสำเร็จ!
```

### Journey C: ผู้ใช้สำรวจ "ทำอะไรได้อีก?"
```
ใช้ไปสักพัก → กดปุ่ม 📖 คู่มือ → แท็บ "ตัวอย่าง"
  ↓
เห็น Use Case ที่ไม่เคยรู้ → กด Copy คำสั่ง
  ↓
วางในแชท → ✅ "ไม่รู้ว่าทำแบบนี้ได้ด้วย!"
```

## 3. เป้าหมาย + ตัวชี้วัด

| เป้าหมาย | ตัวชี้วัด | เกณฑ์ผ่าน |
|---------|---------|---------|
| ผู้ใช้ใหม่เข้าใจระบบ | เวลาจาก สมัคร → อัปโหลดไฟล์แรก | ≤ 5 นาที |
| เชื่อมต่อ MCP สำเร็จ | Copy Config → ทดสอบสำเร็จ | ≥ 80% |
| ค้นพบฟีเจอร์ใหม่ | กดปุ่ม "ลองเลย" ใน Use Case | ≥ 50% |
| ไม่ overwhelm | ผู้ใช้ปิดคู่มือภายใน | ≤ 2 นาที |

## 4. User Stories

| # | ในฐานะ | ฉันต้องการ | เพื่อที่จะ |
|---|--------|-----------|----------|
| US-1 | ผู้ใช้ใหม่ | เปิดเว็บแล้วเห็น Checklist ว่าต้องทำอะไร | ไม่ต้องเดาว่าเริ่มยังไง |
| US-2 | ผู้ใช้ | เห็นรูปภาพประกอบว่าปุ่มอยู่ตรงไหน | กดถูกที่ ไม่ต้องเดา |
| US-3 | ผู้ใช้ | Copy config ได้ทันที ไม่ต้องอ่านยาว | เชื่อมต่อ MCP ใน 2 นาที |
| US-4 | ผู้ใช้ | ดู Use Case ตัวอย่างพร้อมคำสั่ง copy | ค้นพบฟีเจอร์ที่ยังไม่เคยใช้ |
| US-5 | ผู้ใช้ | กดปุ่ม "ลองเลย" แล้วไปหน้านั้นทันที | ไม่ต้องหาเอง |
| US-6 | ผู้ใช้ | พิมพ์ค้นหาในคู่มือ | หาคำตอบเร็ว |

## 5. Functional Requirements

### 5.1 ปุ่มคู่มือ (Guide FAB)

| FR# | คำอธิบาย |
|-----|---------|
| FR-1 | ปุ่ม `📖 คู่มือ` อยู่มุมล่างขวา (FAB, position: fixed) |
| FR-2 | ผู้ใช้ใหม่: ปุ่ม pulse กระพริบ 3 วินาทีเชิญชวนให้กด |
| FR-3 | กดแล้วเปิด Side Drawer ด้านขวา (420px, มือถือ = เต็มจอ) |
| FR-4 | ปิดง่าย: กด ✕ / Escape / คลิกนอก Drawer |

### 5.2 Auto-Onboarding (ผู้ใช้ใหม่)

| FR# | คำอธิบาย |
|-----|---------|
| FR-5 | **ล็อกอินครั้งแรก → คู่มือเปิดอัตโนมัติ** (แสดง Checklist) |
| FR-6 | Checklist: ☐ อัปโหลดไฟล์ / ☐ จัดระเบียบ / ☐ ลองถาม AI / ☐ ต่อ MCP |
| FR-7 | แต่ละรายการเปลี่ยนเป็น ✅ เมื่อทำเสร็จ (ตรวจจาก API) |
| FR-8 | ทำครบ → แสดง 🎉 + ซ่อน Checklist ถาวร |
| FR-9 | จำสถานะไว้ใน `localStorage` — ไม่ auto-show อีก |

### 5.3 แท็บ 1: วิธีใช้งาน (Page Guide) — Progressive Disclosure

| FR# | คำอธิบาย |
|-----|---------|
| FR-10 | **Context-Aware** — แสดงคู่มือของหน้าที่เปิดอยู่อัตโนมัติ |
| FR-11 | **ระดับ 1 (Collapsed):** แสดงแค่ชื่อฟีเจอร์ + ไอคอน ▸ |
| FR-12 | **ระดับ 2 (Expanded):** คลิกแล้วขยาย → เห็นคำอธิบาย + ภาพ |
| FR-13 | แต่ละฟีเจอร์มี: ชื่อ, คำอธิบาย 1-2 บรรทัด, **ภาพ annotated** |
| FR-14 | ปุ่ม **"ลองเลย"** → ปิด Drawer + ไปหน้านั้น |
| FR-15 | Dropdown สลับดูคู่มือหน้าอื่นได้ |

**คู่มือครอบคลุม 8 หน้า:**

| หน้า | ฟีเจอร์ที่อธิบาย |
|------|-----------------|
| 📁 ข้อมูลของฉัน | อัปโหลด, จัดระเบียบทั้งหมด/ใหม่, ดูรายละเอียด |
| 🔍 มุมมองความรู้ | คอลเลกชัน, โน้ต, Context Packs |
| 🕸️ กราฟ | Global/Local, ซูม, คลิกโหนด |
| 💬 AI แชท | ถาม-ตอบ, หลักฐาน, Evidence Graph |
| 👤 โปรไฟล์ | ตัวตน, เป้าหมาย, สไตล์ |
| 🧠 Context Memory | ดู/สร้าง/ลบ/ปักหมุด |
| 🔌 MCP ตั้งค่า | Token, เครื่องมือ, Permissions |
| ⚙️ ตั้งค่า | ภาษา, รหัสผ่าน, บัญชี |

### 5.4 แท็บ 2: เชื่อมต่อแพลตฟอร์ม

```javascript
async function checkOnboardingStatus() {
  const files = await authFetch('/api/files');        // มีไฟล์ไหม?
  const contexts = await authFetch('/api/contexts');   // มี context ไหม?
```

## 8. Acceptance Criteria

### 8.1 ฟังก์ชัน
- [ ] ปุ่ม 📖 คู่มือ อยู่มุมล่างขวาทุกหน้า
- [ ] กดปุ่มแล้ว Drawer เปิด (animated)
- [ ] ผู้ใช้ใหม่: คู่มือเปิดอัตโนมัติ + แสดง Checklist
- [ ] Checklist อัปเดต ✅ เมื่อทำเสร็จ
- [ ] แท็บ "วิธีใช้" = Context-Aware + Progressive Disclosure
- [ ] แท็บ "เชื่อมต่อ" มี 3 แพลตฟอร์ม + Copy Config + ทดสอบ
- [ ] แท็บ "ตัวอย่าง" มี 15+ Use Case + Copy คำสั่ง
- [ ] ค้นหา: พิมพ์แล้ว filter real-time
- [ ] Responsive: มือถือ = เต็มจอ
- [ ] ภาษาไทย 100% (สลับ EN ได้)

### 8.2 UX Metrics
- [ ] ผู้ใช้ใหม่: สมัคร → อัปโหลดไฟล์แรก ≤ 5 นาที
- [ ] Copy Config → เชื่อมต่อสำเร็จ ≥ 80%
- [ ] ผู้ใช้กดปุ่ม "ลองเลย" ≥ 50%
