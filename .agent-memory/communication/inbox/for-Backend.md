## 🛠️ 2. สิ่งที่ฝั่ง Backend ต้องดำเนินการเขียนโค้ด (Action Items)
จากข้อมูลที่ตั้งค่าหน้าบ้านไว้ ทีม Backend จะต้องนำไปประกอบกับแผนการทำงานหลัก (`line-bot-v8.0.0.md`) ดังนี้:

### 📍 งานเร่งด่วน: รองรับคำสั่งจาก Rich Menu
ต้องเข้าไปเพิ่ม Logic ใน `bot_handlers.py` (หรือฟังก์ชัน `handle_text_intent`) เพื่อดักจับและประมวลผลข้อความจากปุ่ม:
- `if text == "สถานะ":` -> คืนค่าข้อมูลโควต้าการใช้งาน และพื้นที่ความจุ
- `if text == "ค้นหา":` -> คืนค่าอธิบายวิธีพิมพ์ค้นหาข้อมูล หรือโยน Flow ไปที่การ Search
- `if text == "ช่วยเหลือ":` -> คืนค่า Flex Message แนะนำวิธีใช้บอท (Help / Onboarding)

### 📍 Phase 3.3: URL Upload from Text
- ตรวจจับ (Regex) เมื่อผู้ใช้พิมพ์ข้อความที่เป็นลิงก์/URL ลงในช่องแชท
- ยิง Quick Reply ถามผู้ใช้ว่าต้องการโหลดข้อมูลจากลิงก์เข้า PDB หรือไม่
- สร้างฟังก์ชันโหลด HTML จากลิงก์ และแปลงเป็น Text เพื่อบันทึกเป็นไฟล์ใหม่

### 📍 Phase 5: Forward File & Edge Cases
- จัดการจังหวะที่ผู้ใช้ Forward ไฟล์เข้ามาในแชททีละหลายๆ ไฟล์ติดกัน (Batch Processing) ไม่ให้ระบบโหลดหนักหรือล่ม
- ดักจับกรณีผู้ใช้ส่งไฟล์เกินโควต้า 10MB
- คืนค่า **Error Flex Message** ทันทีเมื่อมีการส่งประเภทไฟล์ที่ PDB ยังไม่รองรับ (แทนที่จะเป็น Text แข็งๆ)
