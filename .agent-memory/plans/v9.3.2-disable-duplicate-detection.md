## §1 Goal & Why

User สั่ง "ตัดออกก่อนแล้วใช้เบสออน AI แทน" — disable duplicate detection feature ชั่วคราวเพราะ:
1. **Bug ใน [duplicate_detector.py:108](../../backend/duplicate_detector.py#L108)** — `UnicodeEncodeError: surrogates not allowed` ตอน hash text จาก PDF บางตัว · ทำให้ POST `/api/files/{id}/reprocess` ตอบ 500
2. **AI-based system handle implicit อยู่แล้ว** — organizer cluster ไฟล์ semantic similar เข้ากลุ่มเดียวกัน · chat retrieval ใช้ vector search ตาม content · ดังนั้น exact-match dedup เป็น **enhancement** ไม่ใช่ core
3. **User เลือก disable + กลับมาแก้ภายหลัง** — soft disable + TODO marker = revert 1 commit ก็เปิดกลับ

### ลำดับชั้นของ feature ที่จะ disable
- **Layer 1 — Exact match** (`compute_content_hash` + SHA-256 lookup) ← bug ที่นี่
- **Layer 2 — Semantic match** (TF-IDF cosine ≥ 0.80 ผ่าน vector_search)
- **Modal popup** "ไฟล์ซ้ำ — keep/skip" ใน UI
- **Endpoint** `POST /api/files/skip-duplicates`

ทั้งหมด disable ลงไป — frontend จะรับ `duplicates_found: []` เสมอ → modal ไม่ trigger
