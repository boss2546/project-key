## 📚 Context — 5 bugs จาก audit

| ID | Severity | Bug | Impact | Layer |
|----|----------|-----|--------|-------|
| **L1** | 🔴 HIGH | "Connect LINE" button → dead-end | Frontend ส่ง user ไป `/auth/line` ที่ขาด linkToken → error | Frontend (recommend) |
| **L2** | 🔴 HIGH | "Open in LINE" button silent fail | `openLineChat()` อ่าน `window._lineBotUrl` ที่ไม่มีใครเซ็ต | Frontend |
| **L3** | 🔴 HIGH | LINE i18n keys missing 10 keys | TH user toggle → English ติดถาวร | Frontend |
| **L4** | 🟡 MED | Pending linkToken lost on login | sessionStorage เซ็ตไว้ · ไม่มีคนอ่านกลับ | Frontend |
| **L5** | 🟡 MED | applyLanguage ไม่ re-render LINE | badge/notice ใช้ `getLang() ? th : en` ternary in JS | Frontend |

### L1 root cause detail
- `connectLine()` (line_ui.js:89) → POST `/api/line/connect`
- backend (`main.py:1839-1863`): returns `{redirect_url: "{APP_BASE_URL}/auth/line"}` (NO linkToken!)
- `auth-line.js:55`: ตรวจไม่เจอ `linkToken` query → setState error "ไม่พบ linkToken"
- root cause: API design ผิด — Messaging API linkToken ออกได้เฉพาะหลัง user follow bot · ไม่ใช่จาก server-initiated flow

**Fix strategy:** เปลี่ยน frontend ให้เปิด LINE bot URL ตรงๆ (ใช้ `bot_url` จาก `/api/line/status`) แทน POST `/api/line/connect`. Backend endpoint ปล่อยทิ้ง (dead code · ไม่ break old client).

### L4 root cause detail
- `auth-line.js:71`: `sessionStorage.setItem('pdb_pending_line_link', linkToken)` แล้ว redirect `/`
- `landing.js`: 4 login paths · ไม่มี path ไหน read sessionStorage นี้กลับ
- result: linkToken หายตลอด · user ต้องกลับไปเริ่มใหม่จาก LINE bot

**Fix strategy:** เพิ่ม helper `_redirectToPendingLineLink()` · เรียกหลังทุก login path success.

### Step 2 — L1 fix: connectLine opens bot URL directly

**File:** `legacy-frontend/line_ui.js`

**2.1 — Replace `connectLine()` function:**

```js
async function connectLine() {
  // v9.4.2 (L1) — Direct bot URL open instead of dead /api/line/connect endpoint
  // Why: Messaging API linkToken ต้องออกหลัง user follow bot · server-initiated link
  // ทำไม่ได้ — ทาง flow ที่ work: user ใน LINE app → ค้นหา bot → กดเพิ่มเพื่อน
  // → bot ส่ง link prompt → user คลิก → /auth/line?linkToken=xxx
  //
  // Old impl POST /api/line/connect → /auth/line WITHOUT linkToken → error page
  // New impl: เปิด LINE bot URL · user เพิ่มเพื่อน · bot จะ trigger flow ต่อ
  const botUrl = window._lineBotUrl;
  if (!botUrl) {
    showToast(
      getLang() === 'th'
        ? 'LINE bot ยังไม่พร้อมใช้งาน — กรุณารีเฟรชหน้า'
        : 'LINE bot not ready — please refresh the page',
      'error'
    );
    return;
  }

  // เปิด LINE app/web (LINE in-app: deep link · web: HTTPS profile page)
  window.open(botUrl, '_blank');

  // แจ้ง user ว่าต้อง follow bot ใน LINE ก่อน · bot จะส่งลิงก์ confirm กลับมา
  showToast(
    getLang() === 'th'
      ? 'เปิด LINE และเพิ่ม bot เป็นเพื่อน · bot จะส่งลิงก์ยืนยันมาให้'
      : 'Open LINE and add the bot · the bot will send a link prompt',
    'info'
  );
}
```

### Step 5 — L4 fix: pending linkToken redirect after login

**File:** `legacy-frontend/landing.js`

**5.1 — Add helper function (top-level, near other auth helpers):**

```js
// v9.4.2 (L4) — Resume LINE link flow after login
// Why: auth-line.js เซ็ต sessionStorage 'pdb_pending_line_link' เมื่อ user ยังไม่ login
//      แล้ว redirect / · ก่อนรอบนี้ไม่มีใครอ่านกลับ → linkToken หาย → user ต้องเริ่มใหม่จาก LINE bot
// Now: ทุก login success path เรียก helper นี้ · ถ้ามี pending → กลับไป /auth/line?linkToken=xxx
function _redirectToPendingLineLink() {
  const pendingLink = sessionStorage.getItem('pdb_pending_line_link');
  if (!pendingLink) return false;
  sessionStorage.removeItem('pdb_pending_line_link');
  window.location.href = `/auth/line?linkToken=${encodeURIComponent(pendingLink)}`;
  return true;
}
```

## ⚠️ Risks / Open Questions

### Risks (พร้อม mitigation)

**R1 — bot_url = null กรณี LINE_BOT_BASIC_ID ไม่ได้ตั้ง env**
- Backend: `bot_url = None` ถ้า LINE_BOT_BASIC_ID ว่าง
- Frontend: connectLine() check + toast "ยังไม่พร้อม" · ไม่ crash
- Mitigation: accept (server admin จัด config)

**R2 — sessionStorage cross-tab issue**
- ถ้า user เปิด login ใน tab อื่น (หลัง click bot link in LINE in-app browser) → sessionStorage ไม่ share ระหว่าง tab
- Mitigation: localStorage แทน? · NO — security: linkToken มีอายุ 10 นาที · sessionStorage scoped to tab is safer
- Edge case acceptable: rare flow · user can re-trigger from bot

**R3 — i18n keys conflict กับ existing keys**
- Pre-check: grep `'line\.` ใน app.js ไม่มี match → safe to add 10 keys
- Mitigation: verified no conflict

**R4 — Removed /api/line/connect call breaks legacy bookmarks**
- Endpoint ยังคง work ตาม spec · แค่ frontend ไม่เรียก
- Mitigation: ปล่อยทิ้ง (dead code) · clean ในรอบหน้า ถ้าต้อง

**R5 — applyLanguage(lang) loop infinite ถ้า loadLineStatus ตัว trigger applyLanguage?**
- loadLineStatus → fetch /api/line/status → _renderLineStatus → ไม่มี applyLanguage call
- Pattern เดียวกับ renderStorageModeUI / renderDriveErrorBanner · proven safe
- Mitigation: pattern verified · accept
