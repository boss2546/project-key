### Step 5 — P3 JWT warn-log (15 นาที)
**[config.py](../../backend/config.py) แก้ block JWT (line 48-59):**
```python
def _generate_jwt_secret() -> str:
    """Generate and persist a JWT secret key.

    หลังจาก v9.3.0: ถ้า DATA_DIR เป็น mounted volume + ไม่ได้ set JWT_SECRET_KEY env →
    log warning. Production deploy แนะนำ `flyctl secrets set JWT_SECRET_KEY=...` เพราะ:
    - Multi-machine scale: แต่ละ machine generate file ของตัวเอง = JWT mismatch
    - Volume migrate / fork app: file ใหม่ = invalidate session ของ user เก่าทุกคน
    """
    jwt_file = os.path.join(DATA_DIR, ".jwt_secret")
    if os.path.exists(jwt_file):
        with open(jwt_file, "r") as f:
            return f.read().strip()
    secret = secrets.token_urlsafe(64)
    with open(jwt_file, "w") as f:
        f.write(secret)
    return secret


_jwt_env = os.getenv("JWT_SECRET_KEY")
if not _jwt_env and os.path.exists("/app/data"):
    # Production-like deploy (Fly.io mount path) without explicit env var → warn
    import sys
    print(
        "WARN: JWT_SECRET_KEY env var not set in production-like environment. "
        "Falling back to .jwt_secret file in DATA_DIR. This works for single-machine "
        "but breaks multi-machine scale + volume migrate. Set with: "
        "`flyctl secrets set JWT_SECRET_KEY=$(openssl rand -base64 64)`",
        file=sys.stderr,
    )

JWT_SECRET_KEY = _jwt_env or _generate_jwt_secret()
```
**ห้าม fail closed** — ทำ warn-only เพื่อกัน break dev/CI ที่ไม่มี env var

### Step 6 — P4 Drive `invalid_grant` graceful handling (45 นาที)

**[drive_sync.py](../../backend/drive_sync.py) + [storage_router.py](../../backend/storage_router.py):**
1. Import `from google.auth.exceptions import RefreshError`
2. ใน `storage_router.py` เพิ่ม helper:
```python
async def _mark_drive_connection_error(
    db: AsyncSession, connection: DriveConnection, error_msg: str
) -> None:
    """Mark connection as errored — UI จะแสดงปุ่ม Re-connect.

    Why: refresh_token revoke (user เพิกถอนใน Google Account) หรือ client_secret
    rotate ทำให้ refresh fail → ห้าม raise (กัน sync ทุกตัวพังเป็น cascade) →
    mark error + ให้ user re-auth.
    """
    connection.last_sync_status = "error"
    connection.last_sync_error = f"invalid_grant: {error_msg}"[:255]
    await db.commit()
    logger.warning(
        "BYOS: drive refresh failed for user_id=%s — marked error, awaits re-auth",
        connection.user_id,
    )
```
3. Wrap ทุก helper ที่ build credentials → call API ด้วย `try/except RefreshError`:
```python
try:
    creds = build_credentials_from_refresh_token(plaintext)
    drive_service = build("drive", "v3", credentials=creds, cache_discovery=False)
    drive_service.files().list(...).execute()  # หรือ operation อื่น
except RefreshError as e:
    await _mark_drive_connection_error(db, connection, str(e))
    return  # silent — caller ทำงานต่อได้ (best-effort)
```

## §10 Notes for เขียว (gotchas)

1. **CSS cascade order matters** — ในทุก rule fallback chain: `100vh` ต้องมาก่อน · `calc(var(--vh)*100)` ตรงกลาง · `100dvh` สุดท้าย — browsers ที่ support `dvh` จะใช้ตัวสุดท้าย, ที่ไม่ support `dvh` จะ fallback ลงตัวก่อน
2. **IIFE ต้องอยู่ก่อน DOM render** — `app.js` line 1-2 (ก่อนทุกอย่าง) — ห้ามอยู่ใน `DOMContentLoaded` event
3. **Cache-bust หา-แทน** — ใช้ Edit tool replace_all ระวัง: `?v=9.2.2` มีใน comments (`/* v9.2.2 */`) ที่อาจติดเปลี่ยนด้วย — verify diff ก่อน commit
4. **Phase B ใน styles.css** — uncommitted lines 4815+ (cascade rules) **ห้าม commit รวม** patch นี้ · ใช้ `git add -p` เลือกเฉพาะ lines ที่ patch แตะ
5. **P4 catch scope** — เฉพาะ `RefreshError` · ห้าม catch generic `Exception` (กลืน bug จริง) — ดู [google.auth.exceptions](https://google-auth.readthedocs.io/en/latest/reference/google.auth.exceptions.html)
6. **Re-connect button reuse logic** — ใช้ `connectDrive()` function เดิมใน storage_mode.js (no new code path) — ป้องกัน drift
7. **`pricing.html` cache-bust** — ตรวจก่อน sed (อาจมี link asset อื่นที่ไม่ใช่ shared/styles/landing — เช่น Stripe embed)
8. **landing.html load app.js? — YES** verified line 520 — ดังนั้น IIFE ครอบคลุม landing ด้วย ไม่ต้อง duplicate ใน landing.js
