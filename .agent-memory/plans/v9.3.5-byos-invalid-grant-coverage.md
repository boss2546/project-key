## 📡 API Changes

### `POST /api/drive/sync` — Behavioral Change

**ก่อน v9.3.5:**
- HTTP 500 + `Internal Server Error` body เมื่อ token revoked (RefreshError ใน load_connection)
- `drive_connections.last_sync_status` คงเดิม (`"pending"` / สถานะก่อน)
- `last_sync_error = NULL`

**หลัง v9.3.5:**
- HTTP **200** เสมอ (ยกเว้น auth/missing connection ที่ 4xx)
- Response body มี `stats.errors >= 1` เมื่อ RefreshError
- `drive_connections.last_sync_status = "error"`
- `drive_connections.last_sync_error = "RefreshError: invalid_grant: ..."` (truncate 255 chars)

**Response 200 with errors example:**
```json
{
  "status": "completed_with_errors",
  "stats": {
    "pulled_new": 0,
    "pulled_updated": 0,
    "pulled_deleted": 0,
    "pushed_new": 0,
    "pushed_updated": 0,
    "conflicts_resolved": 0,
    "errors": 1,
    "duration_ms": 234
  }
}
```

**Response 200 (success, ไม่เปลี่ยน):**
```json
{
  "status": "ok",
  "stats": { "errors": 0, ... }
}
```

**Errors (ไม่เปลี่ยน):**
- 401 `UNAUTHORIZED` — no JWT
- 400 `NOT_BYOS_MODE` — user ไม่ใช่ byos
- 400 `NO_DRIVE_CONNECTION` — ไม่มี connection row
- 503 `GOOGLE_OAUTH_NOT_CONFIGURED` — env vars ขาด

### `GET /api/drive/status` — No schema change · Coverage extended

`last_sync_error` field ตอนนี้ถูก populate เมื่อ **ทุก** push helper เจอ invalid_grant (ไม่ใช่แค่ profile push เหมือน v9.3.0). UI logic ที่ render "เชื่อมต่อใหม่" ใช้ `last_sync_status='error'` เป็น trigger — มีอยู่แล้ว.

### Step 2 — Wrap `load_connection()` ใน `run_full_sync` ของ `drive_sync.py`

**ปัจจุบัน (line 143-186) — บั๊กที่ load_connection อยู่นอก try:**

```python
async def run_full_sync(self) -> SyncStats:
    if not self._client:
        await self.load_connection()              # ← throws ที่นี่
    assert self._client is not None and self._connection is not None

    started = datetime.utcnow()
    stats: SyncStats = { ... }

    try:                                           # ← try เริ่มที่นี่
        self._connection.last_sync_status = "syncing"
        ...
    except Exception as e:
        ...
```

**หลัง v9.3.5 (ย้าย load_connection เข้า try + handle ก่อน connection):**

```python
async def run_full_sync(self) -> SyncStats:
    started = datetime.utcnow()
    stats: SyncStats = {
        "pulled_new": 0, "pulled_updated": 0, "pulled_deleted": 0,
        "pushed_new": 0, "pushed_updated": 0,
        "conflicts_resolved": 0, "errors": 0, "duration_ms": 0,
    }

    try:
        if not self._client:
            await self.load_connection()  # ← อาจ throw RefreshError ที่ ensure_pdb_folder_structure
        assert self._client is not None and self._connection is not None

        self._connection.last_sync_status = "syncing"
        await self.db.commit()

        await self._push_local_to_drive(stats)
        await self._pull_drive_to_local(stats)

        self._connection.last_sync_status = "success"
        self._connection.last_sync_error = None
        self._connection.last_sync_at = datetime.utcnow()

    except Exception as e:
        # v9.3.5 — load_connection() อาจ throw ก่อน self._connection bind
        # → ลอง re-fetch DriveConnection จาก DB เพื่อ mark error
        from .storage_router import _is_refresh_failure
        if self._connection is not None:
            self._connection.last_sync_status = "error"
            self._connection.last_sync_error = f"{type(e).__name__}: {e}"[:255]
        else:
            # connection ไม่ load สำเร็จ — query DB ตรงๆ เพื่อ mark error
            from sqlalchemy import select as _sel
            res = await self.db.execute(
                _sel(DriveConnection).where(DriveConnection.user_id == self.user_id)
            )
            conn_row = res.scalar_one_or_none()
            if conn_row is not None:
                conn_row.last_sync_status = "error"
                conn_row.last_sync_error = f"{type(e).__name__}: {e}"[:255]

        stats["errors"] += 1
        logger.exception("BYOS sync failed for user %s", self.user_id)
        # ไม่ raise — caller (endpoint) จะเห็น stats.errors > 0

    finally:
        try:
            await self.db.commit()
        except Exception:
            pass
        stats["duration_ms"] = int((datetime.utcnow() - started).total_seconds() * 1000)

    return stats
```

**สำคัญ:**
- Keep `assert` หลัง load_connection เหมือนเดิม (ตอนนี้อยู่ใน try-block — fail early แต่จะถูก except catch)
- `finally` wrap commit ใน try/except เพื่อไม่ให้ commit ตัวสุดท้ายพังงาน return stats

function renderDriveErrorBanner() {
  const banner = document.getElementById('drive-error-banner');

### A. Happy Path (regression — ห้ามพัง)
- **A1** BYOS user upload file (token valid) → `_push_uploads_to_drive` runs → `drive_file_id` set, `storage_source='drive_uploaded'` → `last_sync_status` ไม่เปลี่ยน

- **A4** Managed user `POST /api/drive/sync` → 400 `NOT_BYOS_MODE`

### B. Token Revoked (the bug we're fixing)
- **B1** BYOS user upload file (token revoked) → `_push_uploads_to_drive` → `push_raw_file` ตก except → `_is_refresh_failure(e)=True` → `_mark_drive_connection_errored` commit → `last_sync_status='error'` + `last_sync_error` มี "invalid_grant"
- **B2** BYOS user `POST /api/drive/sync` (token revoked) → **HTTP 200** (ไม่ใช่ 500!) + `status='completed_with_errors'` + `stats.errors >= 1` + `last_sync_status='error'`
- **B3** UI: หลังเรียก `/api/drive/status` → `last_sync_status='error'` → `storage_mode.js renderStorageModeUI` แสดง state error + ปุ่ม "เชื่อมต่อใหม่" + รายละเอียด `last_sync_error` (existing v9.3.0 logic — ไม่แก้)
- **B4** เรียก disconnect + reconnect → `last_sync_status='pending'` (reset ใน OAuth callback line 3292) → upload + push works again

### C. Other failures (กัน false-positive ของ _is_refresh_failure)
- **C1** Drive folder ลบใน Drive UI → upload → push fails with 404 (NOT invalid_grant) → `_is_refresh_failure(e)=False` → ไม่ mark error → log warning เท่านั้น
- **C2** Network offline / Google API down → push fails with timeout → `_is_refresh_failure=False` → ไม่ mark error
- **C3** Quota exceeded ที่ Drive → push fails with 403 quotaExceeded → `_is_refresh_failure=False` → ไม่ mark error

### D. Sync flow specifics
- **D1** Sync with token revoked → `load_connection()` throws RefreshError → caught → mark error via `self._connection` (ตอน load_connection เพิ่งจะ set _connection)
- **D2** Sync with token revoked + connection row missing (race) → load_connection throws ValueError ก่อน DriveClient → except catches → re-query DB → conn_row None → skip mark → return stats with errors=1

## ⚠️ Risks / Open Questions

### Risks (พร้อม mitigation)

**R1 — `_mark_drive_connection_errored` await commit ใน 8 helpers อาจทำให้ background task ช้าลง**
- Impact: Low (~50ms commit per error path)
- Mitigation: error path = bad path ที่ user ต้อง re-auth อยู่แล้ว · acceptable trade-off
- Happy path ไม่ผ่าน branch นี้ → no perf regression

**R2 — Race condition: 2+ background tasks เจอ invalid_grant พร้อมกัน**
- Impact: Medium · 2-3 commits ทับกัน
- Mitigation: SQLite serialize writes · last-write-wins ของ `last_sync_status='error'` ก็ยังถูก
- ไม่มี data loss · ไม่มี state divergence

**R3 — `_is_refresh_failure` heuristic ตกหล่น Google variant**
- Detection: match `RefreshError` class name + `"invalid_grant"` หรือ `"token has been expired or revoked"` substring
- ถ้า Google เปลี่ยน error format → silent fail กลับมา
- Mitigation: log ที่ WARNING level ทุกครั้ง · ฟ้า monitor logs · แก้ heuristic ถ้าเจอ variant ใหม่

**R4 — drive_sync `_pull_drive_to_local` mid-iteration RefreshError**
- ตอนนี้ DriveClient ได้ refresh access_token จาก refresh_token ที่ login. ถ้า Google revoke ระหว่าง iteration → throw mid-loop
- Mitigation: outer try-block ใน `run_full_sync` ครอบคลุมแล้ว → caught by same except → mark error

**R5 — Test fixture coverage**
- ต้อง mock RefreshError ที่ `_build_drive_client` หรือ DriveClient method
- ฟ้าควรใช้ pattern เดียวกับ test เดิมใน `byos_router_smoke.py` ที่ inject mock service
- ถ้า mock ผิดที่ → test ผิด pattern → false positive

### Decisions (แดง pre-decided — user ไม่ต้องตอบเทคนิค · เน้น "best UX, simple design")

**D1 — เพิ่ม endpoint `POST /api/drive/recheck` ไหม?**
- **Decision: NO** — `/api/drive/sync` ทำงานครอบคลุมพอแล้ว · เพิ่ม endpoint = เพิ่ม surface · ฝืน "ง่าย ไม่ซับซ้อน"
- ถ้าวันหนึ่งจำเป็น → เพิ่มใน v9.3.6 หรือ v9.4.x

**D2 — Auto-disconnect หลัง N failures (เช่น 5 ครั้ง) ไหม?**
- **Decision: NO** — destructive · risk false-trigger · UX ตอนนี้ดีพออยู่แล้ว (banner + ปุ่ม "เชื่อมต่อใหม่")
- Auto-disconnect = ระบบลบ connection ของ user เอง = ทำให้ user งงมากกว่าช่วย

**D3 — Submit Google OAuth verification (รื้อ "7-day expire" trap ระยะยาว)?**
- **Decision: ✅ YES — recommend strongly** (เปลี่ยนจาก v1 ที่ default NO)
- **เหตุผล:** ตรงกับ user goal "ใช้งานได้จริง + ดีที่สุด" — ถ้าไม่ verify, user **ทุกคน** จะเจอ token expire ทุก 7 วันตลอดไป (= ดีที่สุดไม่ได้)

### Pattern enforcement
1. **`conn` reference ต้อง bind ก่อน try-block** — ทุก helper มี `_user, conn = pair` แล้ว · อย่าย้ายเข้าใน try
2. **`_is_refresh_failure(e)` เป็น sync function** — call แบบ `if _is_refresh_failure(e):` (ไม่ใช่ await)
3. **`_mark_drive_connection_errored` เป็น async** — ต้อง `await _mark_drive_connection_errored(db, conn, e)`
4. **ลำดับ check:** check `_is_refresh_failure` ก่อน mark · mark ก่อน log · log ก่อน return
5. **Indent level เดิม** — แทรกใน except block ที่มีอยู่ · ห้าม restructure function
